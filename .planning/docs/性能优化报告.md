#### 问题1: 分类删除存在循环查库问题

| 项目 | 内容 |
|------|------|
| **文件路径** | `bms-plugin/bms-plugin-biz/src/main/java/vip/xiaonuo/biz/modular/category/service/impl/BmsCategoryServiceImpl.java` |
| **代码位置** | 第127-136行 `delete()` 方法 |
| **问题描述** | 删除分类时，对每个待删除ID都执行一次数据库查询检查是否存在子分类，存在N次查询问题 |
| **影响级别** | 🔴 高危 |
| **性能影响** | 删除N个分类需要执行N+1次数据库查询，当批量删除大量分类时，数据库压力剧增 |
| **当前代码** | |
| ```java | |
| for (String id : idList) { | |
|     if(this.count(new QueryWrapper<BmsCategory>().lambda().eq(BmsCategory::getParentId, id)) > 0) { | |
|         throw new CommonException("该分类下存在子分类，无法删除"); | |
|     } | |
| } | |
| ``` | |
| **优化建议** | 使用一次查询检查所有待删除ID是否有关联子分类 |
| **优化代码** | |
| ```java | |
| // 优化后：一次查询判断所有待删除分类是否有子分类 | |
| long childCount = this.count(new QueryWrapper<BmsCategory>().lambda() | |
|     .in(BmsCategory::getParentId, idList)); | |
| if (childCount > 0) { | |
|     throw new CommonException("所选分类中存在包含子分类的分类，无法删除"); | |
| } | |
| this.removeByIds(idList); | |
| ``` | |
| **预期收益** | 数据库查询次数从O(N)降低到O(1)，批量删除性能提升90%+ |

#### 问题2: 列表查询无分页限制，存在全表扫描风险

| 项目 | 内容 |
|------|------|
| **文件路径** | 多个ServiceImpl的`list()`方法 |
| **代码位置** | `BmsArticleServiceImpl.java:82-95`、`BmsCommentServiceImpl.java:73-84`、`BmsCategoryServiceImpl.java:76-86` 等 |
| **问题描述** | 所有`list()`方法未添加分页限制，可能返回大量数据 |
| **影响级别** | 🔴 高危 |
| **性能影响** | 评论表预估十万级数据，一次查询可能返回数万条记录，造成内存溢出和数据库压力 |
| **当前代码** | |
| ```java | |
| // BmsCommentServiceImpl.java - 无分页限制 | |
| public List<BmsComment> list(BmsCommentPageParam bmsCommentPageParam) { | |
|     QueryWrapper<BmsComment> queryWrapper = new QueryWrapper<BmsComment>().checkSqlInjection(); | |
|     if(ObjectUtil.isNotEmpty(bmsCommentPageParam.getArticleId())) { | |
|         queryWrapper.lambda().eq(BmsComment::getArticleId, bmsCommentPageParam.getArticleId()); | |
|     } | |
|     queryWrapper.lambda().orderByDesc(BmsComment::getCreateTime); | |
|     return this.list(queryWrapper);  // 可能返回全表数据！ | |
| } | |
| ``` | |
| **优化建议** | 1. 添加默认分页限制 2. 使用分页查询替代列表查询 3. 对于必须返回全量的场景，添加最大条数限制 |
| **优化代码** | |
| ```java | |

|     QueryWrapper<BmsComment> queryWrapper = new QueryWrapper<BmsComment>().checkSqlInjection(); | |

|     queryWrapper.last("LIMIT 1000"); | |
|     return this.list(queryWrapper); | |
| } | |
| ``` | |
| **预期收益** | 避免OOM风险，数据库查询稳定可控 |

#### 问题3: 热点数据未使用缓存

| 项目 | 内容 |
|------|------|
| **文件路径** | 所有业务模块ServiceImpl |
| **代码位置** | 全局 |
| **问题描述** | 分类、标签等基础数据未使用Redis缓存，每次请求都查询数据库 |
| **影响级别** | 🔴 高危 |
| **性能影响** | 分类表、标签表数据量小但访问频繁，是典型热点数据，不缓存会导致不必要的数据库压力 |
| **影响范围** | |
| - 分类数据：前端菜单、文章分类选择等场景频繁使用 | |
| - 标签数据：文章标签选择、标签云展示等场景频繁使用 | |
| **优化建议** | 使用框架提供的`CommonCacheOperator`实现热点数据缓存 |
| **优化代码** | |
| ```java | |
| @Service | |
| public class BmsCategoryServiceImpl extends ServiceImpl<BmsCategoryMapper, BmsCategory> implements BmsCategoryService { | |
|     | |
|     @Resource | |
|     private CommonCacheOperator commonCacheOperator; | |
|     | |
|     private static final String CACHE_KEY_CATEGORY_LIST = "bms:category:list"; | |
|     private static final long CACHE_EXPIRE_SECONDS = 3600; // 1小时过期 | |
|     | |
|     @Override | |
|     public List<BmsCategory> list(BmsCategoryPageParam bmsCategoryPageParam) { | |
|         // 尝试从缓存获取 | |
|         Object cached = commonCacheOperator.get(CACHE_KEY_CATEGORY_LIST); | |
|         if (ObjectUtil.isNotEmpty(cached)) { | |
|             return (List<BmsCategory>) cached; | |
|         } | |
|         | |
|         // 缓存未命中，查询数据库 | |
|         QueryWrapper<BmsCategory> queryWrapper = new QueryWrapper<BmsCategory>().checkSqlInjection(); | |
|         queryWrapper.lambda().eq(BmsCategory::getStatus, BmsCategoryStatusEnum.ENABLE.getValue()); | |
|         queryWrapper.lambda().orderByAsc(BmsCategory::getSortCode); | |
|         List<BmsCategory> list = this.list(queryWrapper); | |
|         | |
|         // 写入缓存 | |
|         commonCacheOperator.put(CACHE_KEY_CATEGORY_LIST, list, CACHE_EXPIRE_SECONDS); | |
|         return list; | |
|     } | |
|     | |
|     @Transactional(rollbackFor = Exception.class) | |
|     @Override | |
|     public void add(BmsCategoryAddParam bmsCategoryAddParam) { | |
|         // ... 业务逻辑 | |
|         this.save(bmsCategory); | |
|         // 清除缓存 | |
|         commonCacheOperator.remove(CACHE_KEY_CATEGORY_LIST); | |
|     } | |
| } | |
| ``` | |
| **预期收益** | 热点数据查询QPS提升10倍以上，数据库压力降低80% |

#### 问题4: LIKE模糊查询可能导致索引失效

| 项目 | 内容 |
|------|------|
| **文件路径** | 多个ServiceImpl的分页查询方法 |
| **代码位置** | `BmsArticleServiceImpl.java:54`、`BmsTagServiceImpl.java:51-54` 等 |
| **问题描述** | 使用`LIKE '%keyword%'`模糊查询，无法使用索引 |
| **影响级别** | 🔴 高危 |
| **性能影响** | 文章表万级数据，模糊查询会进行全表扫描，随着数据增长性能急剧下降 |
| **当前代码** | |
| ```java | |
| // BmsArticleServiceImpl.java | |
| if(ObjectUtil.isNotEmpty(bmsArticlePageParam.getTitle())) { | |
|     queryWrapper.lambda().like(BmsArticle::getTitle, bmsArticlePageParam.getTitle()); | |
| } | |
| ``` | |
| **优化建议** | |
| 1. 对于精确前缀匹配场景，使用`LIKE 'keyword%'`（可使用索引） | |
| 2. 对于全文搜索需求，考虑使用Elasticsearch或MySQL全文索引 | |
| 3. 对于小数据量场景，可接受当前方案 | |
| **优化代码** | |
| ```java | |
| // 方案1：改为前缀匹配（适用于搜索建议等场景） | |
| if(ObjectUtil.isNotEmpty(bmsArticlePageParam.getTitle())) { | |
|     queryWrapper.lambda().likeRight(BmsArticle::getTitle, bmsArticlePageParam.getTitle()); | |
| } | |
| | |
| // 方案2：添加全文索引（适用于大文本搜索） | |
| // SQL: ALTER TABLE BMS_ARTICLE ADD FULLTEXT INDEX FT_TITLE_CONTENT(TITLE, CONTENT); | |
| ``` | |
| **预期收益** | 搜索查询性能提升50%-90%（取决于优化方案） |

#### 问题7: 评论树形结构递归查询风险

| 项目 | 内容 |
|------|------|
| **文件路径** | `bms-plugin/bms-plugin-biz/src/main/java/vip/xiaonuo/biz/modular/comment/service/impl/BmsCommentServiceImpl.java` |
| **代码位置** | 当前实现中未发现树形构建，但数据库设计了PARENT_ID字段 |

| **影响级别** | 🟡 中危 |

| **优化建议** | 采用内存构建树的方式，一次性查询所有评论后在内存中组装 |
| **优化代码示例** | |
| ```java | |
| public List<BmsCommentTreeVO> getCommentTree(String articleId) { | |
|     // 一次性查询该文章所有评论 | |
|     List<BmsComment> allComments = this.list(new QueryWrapper<BmsComment>().lambda() | |
|         .eq(BmsComment::getArticleId, articleId) | |
|         .eq(BmsComment::getStatus, BmsCommentStatusEnum.APPROVED.getValue())); | |
|     | |
|     // 在内存中构建树 | |
|     return buildCommentTree(allComments, "0"); | |
| } | |
| | |
| private List<BmsCommentTreeVO> buildCommentTree(List<BmsComment> allComments, String parentId) { | |
|     return allComments.stream() | |
|         .filter(c -> parentId.equals(c.getParentId())) | |
|         .map(c -> { | |
|             BmsCommentTreeVO vo = BeanUtil.toBean(c, BmsCommentTreeVO.class); | |
|             vo.setChildren(buildCommentTree(allComments, c.getId())); | |
|             return vo; | |
|         }) | |
|         .collect(Collectors.toList()); | |
| } | |
| ``` | |
| **预期收益** | 避免递归数据库查询，评论树构建性能提升90%+ |

#### 问题9: 阅读量/点赞量更新存在并发问题

| 项目 | 内容 |
|------|------|
| **文件路径** | `bms-plugin/bms-plugin-biz/src/main/java/vip/xiaonuo/biz/modular/article/entity/BmsArticle.java` |
| **代码位置** | VIEW_COUNT、LIKE_COUNT字段 |
| **问题描述** | 文章阅读量、点赞量等计数字段，若直接读取-修改-写入，存在并发覆盖问题 |
| **影响级别** | 🟡 中危 |
| **性能影响** | 高并发下计数不准确，且频繁更新同一行记录造成锁竞争 |
| **优化建议** | |
| 1. 使用Redis原子计数器记录增量 | |
| 2. 定时任务同步到数据库 | |
| **优化代码示例** | |
| ```java | |
| // 使用Redis原子计数 | |
| @Service | |
| public class BmsArticleServiceImpl { | |
|     | |
|     @Resource | |
|     private StringRedisTemplate stringRedisTemplate; | |
|     | |
|     private static final String VIEW_COUNT_KEY = "bms:article:view:"; | |
|     | |
|     public void incrementViewCount(String articleId) { | |
|         // Redis原子递增 | |
|         stringRedisTemplate.opsForValue().increment(VIEW_COUNT_KEY + articleId); | |
|     } | |
| } | |
| | |
| // 定时任务同步到数据库 | |
| @Scheduled(fixedRate = 60000) // 每分钟同步一次 | |
| public void syncViewCountToDb() { | |
|     Set<String> keys = stringRedisTemplate.keys(VIEW_COUNT_KEY + "*"); | |
|     for (String key : keys) { | |
|         String articleId = key.replace(VIEW_COUNT_KEY, ""); | |
|         Long count = stringRedisTemplate.opsForValue().getOperations().getExpire(key) > 0 | |
|             ? Long.parseLong(stringRedisTemplate.opsForValue().get(key)) | |
|             : 0L; | |
|         // 更新数据库 | |
|         articleMapper.incrementViewCount(articleId, count); | |
|         // 重置Redis计数 | |
|         stringRedisTemplate.delete(key); | |
|     } | |
| } | |
| ``` | |
| **预期收益** | 解决并发计数问题，降低数据库写压力 |

@Service
public class BmsCategoryServiceImpl extends ServiceImpl<BmsCategoryMapper, BmsCategory> implements BmsCategoryService {

    @Resource
    private CommonCacheOperator commonCacheOperator;

    private static final String CACHE_KEY_CATEGORY_LIST = "bms:category:list";
    private static final String CACHE_KEY_CATEGORY_TREE = "bms:category:tree";
    private static final long CACHE_EXPIRE_SECONDS = 3600;

    @Override
    public List<BmsCategory> list(BmsCategoryPageParam bmsCategoryPageParam) {
        Object cached = commonCacheOperator.get(CACHE_KEY_CATEGORY_LIST);
        if (ObjectUtil.isNotEmpty(cached)) {
            return (List<BmsCategory>) cached;
        }

        QueryWrapper<BmsCategory> queryWrapper = new QueryWrapper<BmsCategory>().checkSqlInjection();
        queryWrapper.lambda().eq(BmsCategory::getStatus, BmsCategoryStatusEnum.ENABLE.getValue());
        queryWrapper.lambda().orderByAsc(BmsCategory::getSortCode);
        List<BmsCategory> list = this.list(queryWrapper);

        commonCacheOperator.put(CACHE_KEY_CATEGORY_LIST, list, CACHE_EXPIRE_SECONDS);
        return list;
    }

    private void clearCategoryCache() {
        commonCacheOperator.remove(CACHE_KEY_CATEGORY_LIST, CACHE_KEY_CATEGORY_TREE);
    }

    @Transactional(rollbackFor = Exception.class)
    @Override
    public void add(BmsCategoryAddParam param) {
        // ... 业务逻辑
        this.save(category);
        clearCategoryCache();
    }

    @Transactional(rollbackFor = Exception.class)
    @Override
    public void edit(BmsCategoryEditParam param) {
        // ... 业务逻辑
        this.updateById(category);
        clearCategoryCache();
    }

    @Transactional(rollbackFor = Exception.class)
    @Override
    public void delete(List<BmsCategoryIdParam> paramList) {
        List<String> idList = CollStreamUtil.toList(paramList, BmsCategoryIdParam::getId);
        if(ObjectUtil.isNotEmpty(idList)) {
            // 优化：一次查询检查子分类
            long childCount = this.count(new QueryWrapper<BmsCategory>().lambda()
                .in(BmsCategory::getParentId, idList));
            if (childCount > 0) {
                throw new CommonException("所选分类中存在包含子分类的分类，无法删除");
            }
            this.removeByIds(idList);
            clearCategoryCache();
        }
    }
}

@Override
public Page<BmsArticle> page(BmsArticlePageParam param) {
    QueryWrapper<BmsArticle> queryWrapper = new QueryWrapper<BmsArticle>().checkSqlInjection();

    // 优化：只SELECT需要的字段
    queryWrapper.select(
        "ID", "TITLE", "SUMMARY", "COVER_IMAGE", "CATEGORY_ID",
        "AUTHOR_ID", "STATUS", "VIEW_COUNT", "LIKE_COUNT", "COMMENT_COUNT",
        "IS_TOP", "IS_RECOMMEND", "PUBLISH_TIME", "CREATE_TIME"
    );

    if(ObjectUtil.isNotEmpty(param.getTitle())) {
        // 优化：使用前缀匹配
        queryWrapper.lambda().likeRight(BmsArticle::getTitle, param.getTitle());
    }
    if(ObjectUtil.isNotEmpty(param.getCategoryId())) {
        queryWrapper.lambda().eq(BmsArticle::getCategoryId, param.getCategoryId());
    }
    if(ObjectUtil.isNotEmpty(param.getStatus())) {
        queryWrapper.lambda().eq(BmsArticle::getStatus, param.getStatus());
    }

    queryWrapper.lambda().orderByDesc(BmsArticle::getIsTop)
        .orderByDesc(BmsArticle::getPublishTime)
        .orderByDesc(BmsArticle::getCreateTime);

    return this.page(CommonPageRequest.defaultPage(), queryWrapper);
}

## 五、总结

BMS博客管理系统的性能问题主要集中在以下几个方面：

1. **数据库层面**：索引设计不完善，缺少组合索引
2. **代码层面**：存在循环查库、全表查询风险
3. **缓存层面**：热点数据未缓存，重复查询数据库
4. **查询优化**：SELECT字段未优化，模糊查询索引失效

建议按照优先级逐步优化，优先解决高危问题，再进行中长期优化。预期优化后：

- 热点查询QPS提升10倍以上
- 列表查询响应时间降低50%以上
- 批量操作性能提升90%以上
- 系统整体稳定性显著提升
